The aim of this project is to design and implement a  bookstore system using REST Web services. Customers access the application via a Web page using their Web browser. 
They can perform two functionalities through that Web page: Looking for books and buying books. 
The system includes three (3) Web services: The Catalog Web service, Payment Web service, and Credit Card Web service.   
Each of the Web services is database-­-backed. The books database contains information about all books in the bookstore. 
The customer accounts database contains information about customers. The credit card accounts database contains customers’ credit card account information:
1. Catalog   Web   Service:   This   service   is   composed   of   two   (2)   operations:    searchBook   and    buyBook.   These   operations   are   invoked   from   the   application’s   Web   page   to   search   for   and   buy    books   respectively.   The   searchBook   operation   accepts   the   first   and   last   names   of   a   given    author   (one   only)   and   returns   the   ISBNs   of   all   books   written   by   that   author.   The   buyBook    operation   accepts   the   ISBNs   of   the   books   to   be   bought   and   customer’s   ID.   We   assume   that   each    customer   is   already   registered   with   the   bookstore   (you   don’t   have   to provide such feature).   The    customer   accounts   database   contains   information   about   each   customer   (ID,   first   name,   last    name,   address,   and   bank   account   information).   During   the   execution   of    buyBook,   the    operation   processPayment   of   the   Payment   Web   service   is   automatically   invoked.   A   response    (“authorized”   or   “unauthorized”)   is   returned   to   the   customer   at   the   end   of   buyBook   execution.       
•   Payment   Web   Service:   This   service   is   composed   of   one   (1)   operation:   processPayment.   The    operation   processPayment   retrieves/stores   the   necessary   information   from   the   customer    accounts   database   and   then,   automatically   invokes   processCreditCard   operation   provided    by   the   Credit   Web   Service.       
•   Credit   Card   Web   Service:   This   service   contains   one   (1)   operation   processCreditCard.   It    returns   “authorized”   or   “unauthorized”   (e.g.,   not   enough   credit,   account   canceled   or   expired, mismatch   in   name,   etc.)
